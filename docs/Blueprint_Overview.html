Sprout — Blueprint Overview

A revival of Microsoft Office 97's animated assistant as a local LLM chat interface, running headless on Raspberry Pi 5. No cloud. No Electron. No display server required.

v0.5.0 Flask + React SPA Pi 5 Ready github.com/bitsandbots/rpi-sprout

01 Quick Start

  1. Install Ollama 
    curl -fsSL https://ollama.com/install.sh | sh
sudo systemctl status ollama
  2. Pull a model (choose based on available RAM — see Tech Stack for recommendations) 
    # 4 GB Pi — either of these
ollama pull llama3.2:1b
ollama pull gemma3:1b

# 8 GB Pi
ollama pull qwen3:4b
  3. Clone the repository 
    git clone https://github.com/bitsandbots/rpi-sprout
cd rpi-sprout
  4. Run the installer — installs dependencies, builds the frontend, registers and starts the systemd service 
    bash install.sh
    Then open http://<pi-ip>:5080 from any device on the LAN. Voice features need Piper voices (optional, post-install): 
    bash scripts/setup_voices.sh        # default voices
bash scripts/setup_voices.sh all    # all available

02 Project Overview

Sprout replaces the original Electron-based Clippy revival with a Flask + React SPA that runs headless on a Raspberry Pi 5, accessible from any browser on the local network. Removing Electron saved ~100 MB of dependencies and eliminated the requirement for a display server.

Core Python Modules

Module Path Responsibility
OllamaService src/python/ollama_service.py LLM inference, model pull fan-out, SSE queues, abort events
ChatManager src/python/chat_manager.py JSON-file CRUD for conversations under ~/.config/Sprout/chats/
SettingsManager src/python/settings_manager.py Dot-notation get/set against settings.json; also hosts DebugSettingsManager
TTSManager src/python/tts_manager.py Piper .onnx voice loading, text → WAV synthesis, voice registry
STTManager src/python/stt_manager.py Faster-Whisper model loading, audio → text (base64 or file)

Thread Safety Pattern

All five managers are Flask-thread-safe singletons using double-checked locking: 

def get_tts_manager() -> TTSManager:
    global _tts
    if _tts is None:
        with _tts_lock:
            if _tts is None:
                _tts = TTSManager()
    return _tts
Resource Lock Protects
_history _history_lock Conversation history (race in streaming + writes)
_loaded_voice _lock Concurrent TTS synthesis
_pull_queues _pull_lock SSE fan-out subscriber list

SSE Streaming Patterns

Two SSE mechanisms are used:

  • LLM inference — each client gets a direct generator. Abort via POST /api/llm/abort sets a threading.Event checked per chunk.
  • Model pull progress — fan-out via queue.Queue per subscriber. Multiple browser tabs can watch the same pull.

Security Controls

Concern Mitigation
Path traversal in chat IDs _valid_chat_id() regex ^[A-Za-z0-9_-]+$
Oversized payloads 10 MB chat write cap, 1 000 message limit, 32 000-char message limit, ~7 MB audio cap
No secrets in code All settings in ~/.config/Sprout/

# 2. Clone and run installer
git clone https://github.com/bitsandbots/rpi-sprout

The installer performs these steps automatically:

  1. Verifies Python 3.11+ and Node.js 20+ are present
  2. Installs system apt packages: libespeak-ng1 libsndfile1 ffmpeg
  3. Installs Python dependencies from requirements.txt
  4. Runs npm install
  5. Builds the React frontend with npm run build → dist/
  6. Generates and installs /etc/systemd/system/sprout.service
  7. Runs systemctl enable --now sprout

05 API Reference

All endpoints prefixed with /api/. Flask server binds 0.0.0.0:5080. Version key in /api/versions response: "sprout". Message sender field: "sprout".

State Management

GET /api/state Full shared state (models + settings)
POST /api/state Set a single setting — body: {"key": "...", "value": ...}
GET /api/debug-state Read debug flags
POST /api/debug-state Write debug flag — body: {"key": "...", "value": ...}

Chat CRUD

GET/api/chatsList all chat records (index, no messages)
GET/api/chats/<chat_id>Get specific chat with all messages
POST/api/chats/<chat_id>Write chat + messages — max 10 MB, 1 000 messages
DELETE/api/chats/<chat_id>Delete a single chat
DELETE/api/chatsDelete all chats

Model Management

GET/api/modelsModel list with download status
POST/api/models/refreshRefresh available model cache
POST /api/models/download Download model — {"tag": "gemma3:1b"} or {"name": "Gemma 3 (1B)"} (tag takes priority)
POST/api/models/deleteDelete model from Ollama — body: {"name": "..."}
POST/api/models/removeRemove from cache without deleting from Ollama
POST/api/models/delete-allDelete all models from Ollama
GET/api/models/pull-progressSSE stream — model download events (fan-out, multiple clients OK)

LLM Session + Streaming

POST/api/llm/createCreate LLM session — model, systemPrompt, temperature, topK
POST/api/llm/destroyDestroy current LLM session
GET/api/llm/streamSSE stream — query params: uuid, message
POST/api/llm/abortAbort in-progress stream — body: {"uuid": "..."}

Ollama Connection

GET/api/ollama/statusConnectivity check — returns URL, connected bool, activeModel
POST/api/ollama/urlChange Ollama base URL — refreshes cache, returns updated model state
GET/api/ollama/discoverScan local subnet for Ollama instances

Voice — TTS + STT

GET/api/voice/stateTTS and STT state, voice list, available STT models
POST/api/voice/tts-toggleToggle TTS — body: {"enabled": true}
POST/api/voice/stt-toggleToggle STT — body: {"enabled": true}
POST/api/voice/set-voiceLoad voice — body: {"voiceId": "en_US-amy-medium"}; persists to settings.json
POST/api/voice/rescanRescan ~/.config/Sprout/voices/ for new voices
POST/api/voice/speakTTS synthesis → audio/wav bytes — body: {"text": "...", "lengthScale": 1.0}
POST/api/voice/transcribeSTT — body: {"audio": "<base64>", "language": "en"}
POST/api/voice/stt-modelChange STT model — body: {"model": "base"}

Utility

GET/api/versionsVersion info — returns {"sprout": "0.5.0", "python": "...", "flask": "..."}

Frontend API Shim

src/renderer/sproutApi.tsx is a backward-compat re-export from api.ts. New code should import directly from api.ts. 

// sproutApi.tsx is a shim — import from api.ts for new code
import { sproutApi, electronAi } from "./sproutApi";

// Get full state
const state = await sproutApi.getFullState();

// Start streaming inference
electronAi.promptStreaming(
  "Hello Sprout",
  { requestUUID: "req-123" },
  {
    onChunk: (text) => console.log(text),
    onDone:  ()     => console.log("done"),
    onError: (err)  => console.error(err),
  }
);

Animation System

Character animations are driven by keywords embedded in LLM responses. Sprout's system prompt instructs the model to include keywords like [Wave] in replies.

Keyword in response Animation
[Wave] Waving hand
[Idle] Standing still
[Thinking] Thinking bubble
[Happy] Happy expression
[Sad] Sad expression
[Listen] Listening pose
⚠
Gotcha: Chat.tsx also handles Key: text colon format as a fallback when the LLM omits bracket syntax. Preserve this fallback if you modify message rendering.

Two animation sets are defined: SPROUT_ANIMATIONS (the custom Sprout character) and CLIPPY_ANIMATIONS (classic Microsoft Clippy). The active character is controlled via Settings → Appearance. Default character: "sprout". Characters are registered in src/renderer/character-animations.tsx.

Animation assets are extracted from assets/animations/ into src/renderer/images/animations/ using tools/extract-animations.sh (run via npm run extract-animations). Only needed when updating source animation assets.

07 Voice Features

Text-to-Speech (TTS)

Piper TTS synthesizes WAV audio from LLM response text. The browser plays audio via the Web Audio API. Voices are .onnx neural models loaded lazily on first use.

Voice ID Gender Description Size
en_US-amy-medium Female Clear American English ~63 MB
en_US-lessac-medium Male Warm American English ~63 MB
en_US-ryan-high Male High-quality, slightly slower ~63 MB
en_US-kusal-medium Male Friendly American English ~63 MB
en_GB-alan-medium Male British English ~63 MB

Speed Control

The POST /api/voice/speak endpoint accepts a lengthScale parameter. Values below 1.0 speed up speech; above 1.0 slow it down. Default is 1.0. See Gap Analysis for UI status.

Custom Voices

  1. Download .onnx and .onnx.json from the Piper repository
  2. Place in ~/.config/Sprout/voices/
  3. Optionally add a .meta.json for display metadata
  4. Click Rescan Voices in Settings → Voice
ℹ
Voice files smaller than 1 KB are skipped automatically (likely a failed partial download). A warning is written to the server console for each skipped file.

Speech-to-Text (STT)

Faster-Whisper with CTranslate2 backend handles transcription. The browser records WebM audio via MediaRecorder, sends it as base64, and the backend converts it to 16 kHz mono WAV via ffmpeg before feeding it to Whisper.

Model Size Accuracy Speed Recommended
tiny ~39 MB Basic Fastest Pi 5 (4 GB)
base ~74 MB Good Fast Pi 5 (4 GB)
small ~243 MB Good Medium Pi 5 (8 GB)
medium ~769 MB High Slow Desktop Linux
large ~1 550 MB Highest Slowest High-RAM only

10 Gap Analysis

These items are documented as planned or partially implemented but are not yet complete. They are tracked as future work.

Hybrid Model Catalog
STATUS: Design approved — backend partially migrated, frontend incomplete

The get_model_state() function is being migrated to return {catalog: {...}, orphans: [...]} instead of a flat model dict, enabling display of models installed in Ollama that aren't in the built-in catalog ("orphans").

Current state: TypeScript types exist (HybridModelState, ManagedModel.actualTag). Python backend is partially migrated. SettingsModel.tsx still reads the flat model dict in parts. ChatContext.tsx uses ollamaTag instead of the resolved actualTag.

Impact: Tag resolution mismatches (e.g., catalog lists qwen3:4b-Q4_K_M but Ollama has qwen3:4b) cause model-not-found errors at inference time. The _resolve_tag() workaround in ollama_service.py mitigates this at the Python layer.

Spec: docs/superpowers/specs/hybrid-model-catalog.md

Docker Deployment
STATUS: Mentioned in docs — untested / unsupported

The installation guide includes a sample Dockerfile with a note that Docker is "not officially supported." Key obstacles: Ollama access requires --network host, and audio device passthrough (/dev/snd) is complex in containers. No CI pipeline validates the Docker build.

Resolution needed: Either validate and document the Docker path fully, or remove the Dockerfile snippet from docs to avoid confusion.

TTS Speed Control UI
STATUS: API complete — no frontend slider

The POST /api/voice/speak endpoint accepts a lengthScale parameter (float, default 1.0). Values below 1.0 speed up speech; above 1.0 slow it down. The TTSManager.synthesize() method accepts length_scale as a parameter.

Missing: SettingsVoice.tsx has no slider for this parameter. Users cannot adjust speaking speed through the UI; the default speed can only be changed by modifying settings_manager.py DEFAULT_SETTINGS.

Character Selection Persistence
STATUS: Needs verification — default set, UI unverified

The application defaults to the "sprout" character (set as DEFAULT_CHARACTER). A character selector exists in Settings → Appearance to switch between Sprout and classic Clippy. The persistence of this choice across page reloads — via settings.json — has not been fully verified.

Characters available: SPROUT_ANIMATIONS (custom green-themed character) and CLIPPY_ANIMATIONS (classic). Registered in src/renderer/character-animations.tsx.
